Neste documento descreve-se o programa que gera os relatórios individuais de avaliação por participante.

Descrição do programa de geração de relatórios individuais

O programa Alcaide gera relatórios individuais detalhados por participante, a partir dos ficheiros de resultados produzidos pelo programa Sultão, descrito na página de documentação dos programas do HAREM clássico. O programa cria uma directoria por cada sistema, onde escreve o relatório e coloca as figuras correspondentes. Na directoria raiz cria um ficheiro index.html a partir do qual se pode ver todos os relatórios.

O programa está configurado para ler os seguintes ficheiros de resultados:

  • Resultados do HAREM clássico para os cenários de avaliação, incluindo de categorias: resultados[_altrel]_<cenario>.html e resultados[_altrel]_<categoria>.html
  • Resultados dos vários modos da pista TEMPO, incluindo os do HAREM clássico na mini CD do TEMPO, para os cenário de avaliação do TEMPO estendido: resultadosTEMPO_classico_<cenario>.html e resultadosTEMPO_estendido_<cenario>_{completo|sonorm|semnorm}.html

Invocação:

./Alcaide.r [-dresultados directoria1] [-drelatorios directoria2]

Argumentos opcionais:

  • -dresultados <directoria1>: indica a directoria onde se encontram os ficheiros de resultados produzidos pelo programa Sultão; caso seja omitida, o programa tentará ler os ficheiros da directoria resultados a partir da directoria de onde se estiver a invocar o programa;
  • -drelatorios <directoria2>: indica a directoria raiz onde serão criadas as directorias, uma por cada sistema, que conterão o relatório do sistema e os gráficos produzidos; caso seja omitida, o programa criará as directorias na directoria resultados/relatorios a partir da directoria de onde se estiver a invocar o programa.

Integração com programas de avaliação

Este programa deve ser usado no fim da sequência completa de invocação dos programas de avaliação para o HAREM clássico e TEMPO estendido, nos vários cenários de avaliação, incluindo por categorias. O ficheiro Avaliacao.sh ilustra uma sequência completa de invocação dos programas com geração de relatórios.

Informações adicionais sobre o R

O programa foi escrito em R e testado tanto no Darwin como no ambiente R para MacOSX. Não foram utilizados pacotes adicionais do R, mas a distribuição de R tem de ter sido compilada para suportar png e o X11 deve estar instalado na máquina. Para informações sobre como instalar, consultar a página do projecto R.
